Концепции — как работает авторизация
🔑

JWT — как работает

JSON Web Token — самый популярный способ авторизации в SPA. Сервер выдаёт два токена: access (короткоживущий) и refresh (долгоживущий).

Схема работы

1️⃣Пользователь вводит логин/пароль → POST /login
2️⃣Сервер возвращает access_token (15 мин) + refresh_token (7 дней)
3️⃣Клиент отправляет access_token в каждом запросе в заголовке
4️⃣access_token истёк → клиент отправляет refresh_token → получает новый access_token
5️⃣refresh_token истёк → разлогинить, перейти на /login

Структура JWT токена

📌Header — алгоритм подписи (HS256)
📌Payload — данные: userId, role, exp (время истечения)
📌Signature — подпись, проверяется на сервере
JWT можно декодировать без ключа — не храни в нём секреты (пароль, карты). Только публичные данные: id, роль.

Где хранить токены — сравнение

ХранилищеПлюсыМинусыРекомендация
localStorageПросто, сохраняется между сессиямиУязвим к XSS — скрипт может украсть токенПриемлемо для несекретных приложений
sessionStorageОчищается при закрытии вкладкиТоже уязвим к XSSНе рекомендуется
Cookie (httpOnly)Недоступен для JS — защита от XSSТребует настройки CORS, уязвим к CSRFЛучший вариант для production
Memory (переменная)Максимальная защитаТеряется при обновлении страницыДля access_token + refresh в cookie
Лучший паттерн: access_token в памяти (переменная/state), refresh_token в httpOnly cookie. Если не настраиваешь backend — localStorage приемлем для учебных проектов.
Redux — authSlice и authApi
🏪

authSlice.ts — хранение состояния авторизации

Slice хранит токен и данные пользователя. RTK Query делает запросы к API.

features/auth/authSlice.ts

import { createSlice, PayloadAction } from '@reduxjs/toolkit'

interface User {
  id:    number
  name:  string
  email: string
  role:  'admin' | 'user' | 'guest'
}

interface AuthState {
  token: string | null
  user:  User   | null
}

const initialState: AuthState = {
  // Читаем токен из localStorage при старте приложения
  token: localStorage.getItem('token'),
  user:  null,
}

const authSlice = createSlice({
  name: 'auth',
  initialState,
  reducers: {
    setCredentials: (state, action: PayloadAction<{ token: string; user: User }>) => {
      state.token = action.payload.token
      state.user  = action.payload.user
      localStorage.setItem('token', action.payload.token)
    },
    setUser: (state, action: PayloadAction<User>) => {
      state.user = action.payload
    },
    logout: (state) => {
      state.token = null
      state.user  = null
      localStorage.removeItem('token')
    },
  },
})

export const { setCredentials, setUser, logout } = authSlice.actions
export default authSlice.reducer

// Селекторы
export const selectToken   = (state: RootState) => state.auth.token
export const selectUser    = (state: RootState) => state.auth.user
export const selectIsAuth  = (state: RootState) => Boolean(state.auth.token)
export const selectIsAdmin = (state: RootState) => state.auth.user?.role === 'admin'

features/auth/authApi.ts — RTK Query эндпоинты

import { createApi, fetchBaseQuery } from '@reduxjs/toolkit/query/react'
import type { RootState } from '../../app/store'

interface LoginRequest    { email: string; password: string }
interface RegisterRequest { name: string; email: string; password: string }
interface AuthResponse    { token: string; user: User }

export const authApi = createApi({
  reducerPath: 'authApi',
  baseQuery: fetchBaseQuery({
    baseUrl: import.meta.env.VITE_API_URL,
    // Автоматически добавляет токен ко всем запросам
    prepareHeaders: (headers, { getState }) => {
      const token = (getState() as RootState).auth.token
      if (token) {
        headers.set('Authorization', `Bearer ${token}`)
      }
      return headers
    },
  }),
  endpoints: (builder) => ({

    // Вход
    login: builder.mutation<AuthResponse, LoginRequest>({
      query: (credentials) => ({
        url:    '/auth/login',
        method: 'POST',
        body:   credentials,
      }),
    }),

    // Регистрация
    register: builder.mutation<AuthResponse, RegisterRequest>({
      query: (data) => ({
        url:    '/auth/register',
        method: 'POST',
        body:   data,
      }),
    }),

    // Получить текущего пользователя (при старте приложения)
    getMe: builder.query<User, void>({
      query: () => '/auth/me',
    }),

    // Выход
    logoutApi: builder.mutation<void, void>({
      query: () => ({ url: '/auth/logout', method: 'POST' }),
    }),

  }),
})

export const {
  useLoginMutation,
  useRegisterMutation,
  useGetMeQuery,
  useLogoutApiMutation,
} = authApi
authApi и authSlice работают вместе: authApi делает запросы, authSlice хранит результат. После успешного login — вызываем dispatch(setCredentials(...)) чтобы сохранить токен в store.

shared/lib/axios.ts — интерцепторы авторизации

Axios автоматически добавляет токен ко всем запросам и обновляет его при истечении — без изменений в компонентах.

Базовый интерцептор — добавить токен

import axios from 'axios'
import { store } from '../../app/store'
import { logout } from '../../features/auth/authSlice'

export const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  timeout: 10_000,
})

// REQUEST — добавляем токен к каждому запросу
api.interceptors.request.use((config) => {
  const token = store.getState().auth.token
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// RESPONSE — обрабатываем ошибки глобально
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // Токен невалиден — разлогиниваем
      store.dispatch(logout())
      window.location.href = '/login'
    }
    return Promise.reject(error)
  }
)

Продвинутый вариант — авто-обновление access_token

import axios, { AxiosError } from 'axios'
import { store } from '../../app/store'
import { setCredentials, logout } from '../../features/auth/authSlice'

export const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  timeout: 10_000,
})

// REQUEST — добавляем токен
api.interceptors.request.use((config) => {
  const token = store.getState().auth.token
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// Флаг — идёт ли сейчас обновление токена
let isRefreshing = false
// Очередь запросов которые ждут новый токен
let refreshQueue: Array<(token: string) => void> = []

// RESPONSE — перехватываем 401 и обновляем токен
api.interceptors.response.use(
  (response) => response,
  async (error: AxiosError) => {
    const original = error.config as any

    // Если 401 и это не повторный запрос
    if (error.response?.status === 401 && !original._retry) {
      original._retry = true

      if (isRefreshing) {
        // Уже обновляем — ставим запрос в очередь
        return new Promise((resolve) => {
          refreshQueue.push((token) => {
            original.headers.Authorization = `Bearer ${token}`
            resolve(api(original))
          })
        })
      }

      isRefreshing = true

      try {
        // Запрашиваем новый токен
        const { data } = await axios.post(
          `${import.meta.env.VITE_API_URL}/auth/refresh`,
          {},
          { withCredentials: true }  // refresh_token в httpOnly cookie
        )

        const newToken = data.token

        // Сохраняем новый токен
        store.dispatch(setCredentials({ token: newToken, user: data.user }))

        // Повторяем все запросы из очереди с новым токеном
        refreshQueue.forEach((cb) => cb(newToken))
        refreshQueue = []

        // Повторяем исходный запрос
        original.headers.Authorization = `Bearer ${newToken}`
        return api(original)

      } catch {
        // Refresh не удался — разлогиниваем
        store.dispatch(logout())
        refreshQueue = []
        window.location.href = '/login'
        return Promise.reject(error)
      } finally {
        isRefreshing = false
      }
    }

    return Promise.reject(error)
  }
)
isRefreshing + очередь — ключевой момент. Без этого при одновременных запросах с истёкшим токеном каждый из них попытается обновить токен отдельно, что приводит к ошибкам.

Форма регистрации — полный пример

react-hook-form + Zod + RTK Query mutation + Redux dispatch. Все части стека вместе.

pages/RegisterPage.tsx

import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'
import { useNavigate } from 'react-router-dom'
import { useDispatch } from 'react-redux'
import { useRegisterMutation } from '../features/auth/authApi'
import { setCredentials } from '../features/auth/authSlice'

// 1. Схема валидации
const schema = z.object({
  name: z.string()
    .min(2, 'Имя минимум 2 символа')
    .max(50, 'Имя максимум 50 символов'),

  email: z.string()
    .email('Введите корректный email'),

  password: z.string()
    .min(8, 'Пароль минимум 8 символов')
    .regex(/[A-Z]/, 'Должна быть хотя бы одна заглавная буква')
    .regex(/[0-9]/, 'Должна быть хотя бы одна цифра'),

  confirmPassword: z.string(),
})
.refine(
  (data) => data.password === data.confirmPassword,
  { message: 'Пароли не совпадают', path: ['confirmPassword'] }
)

type RegisterFormData = z.infer<typeof schema>

export default function RegisterPage() {
  const navigate  = useNavigate()
  const dispatch  = useDispatch()

  const [register, { isLoading }] = useRegisterMutation()

  const {
    register: field,
    handleSubmit,
    setError,
    formState: { errors },
  } = useForm<RegisterFormData>({ resolver: zodResolver(schema) })

  const onSubmit = async (data: RegisterFormData) => {
    try {
      const result = await register({
        name:     data.name,
        email:    data.email,
        password: data.password,
      }).unwrap()

      // Сохраняем токен и пользователя в Redux
      dispatch(setCredentials({ token: result.token, user: result.user }))

      // Переходим на главную
      navigate('/', { replace: true })

    } catch (err: any) {
      // Ошибка от сервера — email уже занят
      if (err?.data?.field === 'email') {
        setError('email', { message: err.data.message })
      }
    }
  }

  return (
    <div className="max-w-md mx-auto mt-16 p-8 bg-gray-900 rounded-xl">
      <h1 className="text-2xl font-bold mb-6">Регистрация</h1>

      <form onSubmit={handleSubmit(onSubmit)} className="flex flex-col gap-4">

        <div>
          <input
            {...field('name')}
            placeholder="Имя"
            className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg"
          />
          {errors.name && <p className="text-red-400 text-sm mt-1">{errors.name.message}</p>}
        </div>

        <div>
          <input
            {...field('email')}
            type="email"
            placeholder="Email"
            className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg"
          />
          {errors.email && <p className="text-red-400 text-sm mt-1">{errors.email.message}</p>}
        </div>

        <div>
          <input
            {...field('password')}
            type="password"
            placeholder="Пароль"
            className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg"
          />
          {errors.password && <p className="text-red-400 text-sm mt-1">{errors.password.message}</p>}
        </div>

        <div>
          <input
            {...field('confirmPassword')}
            type="password"
            placeholder="Подтвердите пароль"
            className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg"
          />
          {errors.confirmPassword && <p className="text-red-400 text-sm mt-1">{errors.confirmPassword.message}</p>}
        </div>

        <button
          type="submit"
          disabled={isLoading}
          className="w-full py-2 bg-blue-600 hover:bg-blue-700 rounded-lg font-semibold disabled:opacity-50"
        >
          {isLoading ? 'Регистрация...' : 'Зарегистрироваться'}
        </button>

      </form>

      <p className="mt-4 text-gray-400 text-sm text-center">
        Уже есть аккаунт?&nbsp;
        <a href="/login" className="text-blue-400 hover:underline">Войти</a>
      </p>
    </div>
  )
}

pages/LoginPage.tsx

import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import { z } from 'zod'
import { useNavigate, useSearchParams } from 'react-router-dom'
import { useDispatch } from 'react-redux'
import { useLoginMutation } from '../features/auth/authApi'
import { setCredentials } from '../features/auth/authSlice'

const schema = z.object({
  email:    z.string().email('Введите корректный email'),
  password: z.string().min(1, 'Введите пароль'),
})

type LoginFormData = z.infer<typeof schema>

export default function LoginPage() {
  const navigate      = useNavigate()
  const dispatch      = useDispatch()
  const [params]      = useSearchParams()

  const [login, { isLoading }] = useLoginMutation()

  const { register, handleSubmit, setError, formState: { errors } } =
    useForm<LoginFormData>({ resolver: zodResolver(schema) })

  const onSubmit = async (data: LoginFormData) => {
    try {
      const result = await login(data).unwrap()
      dispatch(setCredentials({ token: result.token, user: result.user }))

      // Возвращаем на страницу откуда пришли (или на главную)
      const returnTo = params.get('returnTo') ?? '/'
      navigate(returnTo, { replace: true })

    } catch (err: any) {
      if (err?.status === 401) {
        setError('password', { message: 'Неверный email или пароль' })
      }
    }
  }

  return (
    <div className="max-w-md mx-auto mt-16 p-8 bg-gray-900 rounded-xl">
      <h1 className="text-2xl font-bold mb-6">Вход</h1>

      <form onSubmit={handleSubmit(onSubmit)} className="flex flex-col gap-4">
        <div>
          <input {...register('email')} type="email" placeholder="Email"
            className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg" />
          {errors.email && <p className="text-red-400 text-sm mt-1">{errors.email.message}</p>}
        </div>

        <div>
          <input {...register('password')} type="password" placeholder="Пароль"
            className="w-full px-3 py-2 bg-gray-800 border border-gray-700 rounded-lg" />
          {errors.password && <p className="text-red-400 text-sm mt-1">{errors.password.message}</p>}
        </div>

        <button type="submit" disabled={isLoading}
          className="w-full py-2 bg-blue-600 hover:bg-blue-700 rounded-lg font-semibold disabled:opacity-50">
          {isLoading ? 'Входим...' : 'Войти'}
        </button>
      </form>

      <p className="mt-4 text-gray-400 text-sm text-center">
        Нет аккаунта?&nbsp;
        <a href="/register" className="text-blue-400 hover:underline">Зарегистрироваться</a>
      </p>
    </div>
  )
}
Защита маршрутов — BrowserRouter и createBrowserRouter
🛡️

BrowserRouter — ProtectedRoute компонент

Классический способ защиты маршрутов. Компонент-обёртка проверяет авторизацию и редиректит при необходимости.

components/ProtectedRoute.tsx

import { Navigate, Outlet, useLocation } from 'react-router-dom'
import { useAppSelector } from '../app/hooks'
import { selectIsAuth, selectUser } from '../features/auth/authSlice'

interface ProtectedRouteProps {
  requiredRole?: 'admin' | 'user'
}

export default function ProtectedRoute({ requiredRole }: ProtectedRouteProps) {
  const isAuth   = useAppSelector(selectIsAuth)
  const user     = useAppSelector(selectUser)
  const location = useLocation()

  // Не авторизован — редирект на /login с сохранением текущего URL
  if (!isAuth) {
    return <Navigate to={`/login?returnTo=${location.pathname}`} replace />
  }

  // Нет нужной роли — редирект на /403
  if (requiredRole && user?.role !== requiredRole) {
    return <Navigate to="/403" replace />
  }

  return <Outlet />
}

// App.tsx — использование
<Routes>
  {/* Публичные маршруты */}
  <Route path="/login"    element={<LoginPage />} />
  <Route path="/register" element={<RegisterPage />} />

  {/* Защищённые — для всех авторизованных */}
  <Route element={<ProtectedRoute />}>
    <Route path="/"         element={<HomePage />} />
    <Route path="/profile"  element={<ProfilePage />} />
    <Route path="/settings" element={<SettingsPage />} />
  </Route>

  {/* Только для admin */}
  <Route element={<ProtectedRoute requiredRole="admin" />}>
    <Route path="/admin" element={<AdminPage />} />
  </Route>

  <Route path="/403" element={<Forbidden />} />
  <Route path="*"    element={<NotFound />} />
</Routes>

createBrowserRouter — защита через loader

Современный способ. Редирект происходит до рендера компонента — пользователь не увидит даже вспышки защищённой страницы.

router.tsx — защита через loader

import { createBrowserRouter, redirect } from 'react-router-dom'
import { store } from './app/store'
import { selectIsAuth, selectUser } from './features/auth/authSlice'

// Вспомогательные функции проверки
function requireAuth(request: Request) {
  const isAuth = selectIsAuth(store.getState())
  if (!isAuth) {
    const url      = new URL(request.url)
    const returnTo = encodeURIComponent(url.pathname + url.search)
    throw redirect(`/login?returnTo=${returnTo}`)
  }
}

function requireRole(role: string) {
  requireAuth(new Request(window.location.href))
  const user = selectUser(store.getState())
  if (user?.role !== role) throw redirect('/403')
}

export const router = createBrowserRouter([

  // Публичные маршруты
  {
    path: '/login',
    element: <LoginPage />,
    loader: () => {
      // Уже залогинен — перенаправляем на главную
      if (selectIsAuth(store.getState())) return redirect('/')
      return null
    },
  },
  { path: '/register', element: <RegisterPage /> },

  // Защищённые маршруты — один loader на всю группу
  {
    id: 'protected',
    element: <RootLayout />,
    loader: ({ request }) => {
      requireAuth(request)             // проверяем авторизацию
      return null                      // можно загрузить данные пользователя
    },
    children: [
      { index: true,        element: <HomePage /> },
      { path: 'profile',    element: <ProfilePage /> },
      { path: 'settings',   element: <SettingsPage /> },
      {
        path: 'admin',
        element: <AdminPage />,
        loader: () => { requireRole('admin'); return null },
      },
    ],
  },

  { path: '/403', element: <Forbidden /> },
  { path: '*',    element: <NotFound /> },
])
requireAuth читает store напрямую через store.getState() — без хуков. Это работает потому что loader выполняется вне React-дерева.
Инициализация — загрузка пользователя при старте
⚡

Загрузить данные пользователя при старте приложения

При обновлении страницы токен есть в localStorage, но данные пользователя (имя, роль) нужно загрузить заново с сервера.

App.tsx — инициализация при монтировании

import { useEffect } from 'react'
import { useAppDispatch, useAppSelector } from './app/hooks'
import { selectToken, setUser, logout } from './features/auth/authSlice'
import { useGetMeQuery } from './features/auth/authApi'

function AppInitializer({ children }: { children: React.ReactNode }) {
  const token = useAppSelector(selectToken)

  // Загружаем профиль только если есть токен
  const { data: user, isError } = useGetMeQuery(undefined, {
    skip: !token,  // не делать запрос если токена нет
  })

  const dispatch = useAppDispatch()

  useEffect(() => {
    if (user)    dispatch(setUser(user))
    if (isError) dispatch(logout())  // токен невалиден — разлогинить
  }, [user, isError, dispatch])

  return <>{children}</>
}

// main.tsx
<Provider store={store}>
  <AppInitializer>
    <RouterProvider router={router} />
  </AppInitializer>
</Provider>
Этот паттерн решает проблему: токен есть, но после обновления страницы user = null. AppInitializer запрашивает /auth/me при старте и восстанавливает данные пользователя.
